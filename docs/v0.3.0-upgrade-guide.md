# Upgrade Guide: v0.2.x to v0.3.0

## Region Key Normalization (Breaking Change)

Starting in v0.3.0, the module normalizes all `for_each` keys derived from region strings to lowercase AWS format (`us-east-1`). Previously, the module used the raw user-provided region string as the key.

### Who is affected?

Only users who passed **Atlas-format regions** (`US_EAST_1`) to these variables:
- `privatelink_endpoints[].region`
- `encryption.private_endpoint_regions`

If you already use lowercase AWS format (`us-east-1`) in your configuration, **no action is required**.

### How to check

After upgrading the module source to v0.3.0, run `terraform plan`. If you see Terraform destroying and recreating resources because of the region key format change, the update affects your configuration:

```
# module.atlas_aws.mongodbatlas_privatelink_endpoint.this["US_EAST_1"] will be destroyed
# module.atlas_aws.mongodbatlas_privatelink_endpoint.this["us-east-1"] will be created
```

### Migration with `moved` blocks (recommended)

Add `moved` blocks in your root module for each Atlas-format region you used. Replace `module.atlas_aws` with your actual module name.

**PrivateLink endpoints** -- two `moved` blocks per region:

```terraform
moved {
  from = module.atlas_aws.mongodbatlas_privatelink_endpoint.this["US_EAST_1"]
  to   = module.atlas_aws.mongodbatlas_privatelink_endpoint.this["us-east-1"]
}

moved {
  from = module.atlas_aws.module.privatelink["US_EAST_1"]
  to   = module.atlas_aws.module.privatelink["us-east-1"]
}
```

**Encryption private endpoints** -- one `moved` block per region:

```terraform
moved {
  from = module.atlas_aws.module.encryption_private_endpoint["US_EAST_1"]
  to   = module.atlas_aws.module.encryption_private_endpoint["us-east-1"]
}
```

After adding the `moved` blocks:

1. Run `terraform plan` -- the plan should show no changes
2. Run `terraform apply` -- Terraform updates the state in-place
3. Remove the `moved` blocks from your configuration (Terraform ignores resolved `moved` blocks, but removing them keeps your code clean)

### Migration with `terraform state mv` (alternative)

If you prefer not to add `moved` blocks, use `terraform state mv` commands directly. Replace `module.atlas_aws` with your actual module name and `US_EAST_1` with each Atlas-format region you used:

```bash
terraform state mv \
  'module.atlas_aws.mongodbatlas_privatelink_endpoint.this["US_EAST_1"]' \
  'module.atlas_aws.mongodbatlas_privatelink_endpoint.this["us-east-1"]'

terraform state mv \
  'module.atlas_aws.module.privatelink["US_EAST_1"]' \
  'module.atlas_aws.module.privatelink["us-east-1"]'

terraform state mv \
  'module.atlas_aws.module.encryption_private_endpoint["US_EAST_1"]' \
  'module.atlas_aws.module.encryption_private_endpoint["us-east-1"]'
```

### Output key changes

Output map keys also change from Atlas format to AWS format:
- `module.atlas_aws.privatelink["US_EAST_1"]` becomes `module.atlas_aws.privatelink["us-east-1"]`
- `module.atlas_aws.privatelink_service_info["US_EAST_1"]` becomes `module.atlas_aws.privatelink_service_info["us-east-1"]`
- `module.atlas_aws.encryption.private_endpoints["US_EAST_1"]` becomes `module.atlas_aws.encryption.private_endpoints["us-east-1"]`

Update any references to these outputs in your configuration.

### Full example

Before (v0.2.x):

```terraform
module "atlas_aws" {
  source     = "terraform-mongodbatlas-modules/atlas-aws/mongodbatlas"
  version    = "~> 0.2"
  project_id = var.project_id

  privatelink_endpoints = [
    { region = "US_EAST_1", subnet_ids = [aws_subnet.east.id] },
    { region = "US_WEST_2", subnet_ids = [aws_subnet.west.id] },
  ]

  encryption = {
    enabled                  = true
    kms_key_arn              = aws_kms_key.atlas.arn
    private_endpoint_regions = ["US_EAST_1"]
  }
}

resource "aws_instance" "app" {
  # referencing output by Atlas-format key
  tags = {
    pl_status = module.atlas_aws.privatelink["US_EAST_1"].status
  }
}
```

After (v0.3.0):

```terraform
module "atlas_aws" {
  source     = "terraform-mongodbatlas-modules/atlas-aws/mongodbatlas"
  version    = "~> 0.3"
  project_id = var.project_id

  privatelink_endpoints = [
    { region = "US_EAST_1", subnet_ids = [aws_subnet.east.id] },
    { region = "US_WEST_2", subnet_ids = [aws_subnet.west.id] },
  ]

  encryption = {
    enabled                  = true
    kms_key_arn              = aws_kms_key.atlas.arn
    private_endpoint_regions = ["US_EAST_1"]
  }
}

resource "aws_instance" "app" {
  # output keys are now lowercase AWS format regardless of input format
  tags = {
    pl_status = module.atlas_aws.privatelink["us-east-1"].status
  }
}

# Temporary moved blocks -- remove after first successful apply
moved {
  from = module.atlas_aws.mongodbatlas_privatelink_endpoint.this["US_EAST_1"]
  to   = module.atlas_aws.mongodbatlas_privatelink_endpoint.this["us-east-1"]
}
moved {
  from = module.atlas_aws.module.privatelink["US_EAST_1"]
  to   = module.atlas_aws.module.privatelink["us-east-1"]
}
moved {
  from = module.atlas_aws.mongodbatlas_privatelink_endpoint.this["US_WEST_2"]
  to   = module.atlas_aws.mongodbatlas_privatelink_endpoint.this["us-west-2"]
}
moved {
  from = module.atlas_aws.module.privatelink["US_WEST_2"]
  to   = module.atlas_aws.module.privatelink["us-west-2"]
}
moved {
  from = module.atlas_aws.module.encryption_private_endpoint["US_EAST_1"]
  to   = module.atlas_aws.module.encryption_private_endpoint["us-east-1"]
}
```

The input `region` values can stay in either format -- the module normalizes internally. Only the `for_each` keys (state addresses) and output map keys change.
